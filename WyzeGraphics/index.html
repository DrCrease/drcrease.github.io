Hey, Wyze Fans!

This HTML isn’t pretty, but it’s not meant to be. (I didn’t cook up any super fancy CSS to go with it.) Hopefully the graphics look better and/or are more useful than this garbage markup. You can expand the sections below to look at the pictures and decide for yourself. Because the images are SVGs, they should scale in each section/row to make use of the available page width, and you can click an individual image to see it all blowed up an’ whatnot.

Feel free to steal these SVGs and use as you will for promoting or showing your support for Wyze. (Don’t be a jerk with these or use them for nefarious purposes.) I hand-coded them (using the source noted in the comment of each as a starting point) rather than using a GUI editor and tried to comment along the way so that hopefully any necessary future modifications should be relatively straightforward. There might still be some extraneous code in them because of that, but it shouldn’t cause any problems or junk them up like a graphical program might.

Because of that superfluous code, please note that the availability of these images is subject to change and that I may do some additional clean-up/refinement with these at some point. For instance, while translating these from CodePen and preparing them as stand-alone SVGs, I noticed that there are some unnecessary line commands in the paths, and many of the paths themselves are a mix of relative and absolute coordinates, which just seems messy. (I was just experimenting and learning to write SVGs—and, specifically, SVG animation—when I created these, so gimme a break, eh?) Anyway, if you do use them, then please contact me via personal message at Wyze Forum. I’d like to know how/if these are being used and also be able to update any users if/when I get around to doing clean-up.
